# F-TEMPO data ingest pipeline

## Data Discovery & Download

We can download data from these libraries:

 * GB-Lbl (British Library)
 * D-MBS: Bayerische Staatsbibliothek, München

    python download_library.py metadatafile library_id download_dir

where metadatafile is a csv file with columns `rismid,manifest`. `manifest` is a URL to a IIIF manifest.

    python download_library gblbl-meta.csv GB-Lbl ../data-download

This will create a directory structure `download_dir/GB-Lbl/[manifest_id]/`
Within this directory we create:

 * manifest.json: the IIIF manifest file that we downloaded
 * rism-source.json: The data from RISM online for this RISM id
 * other files per library, e.g. GB-Lbl will collect marc.json (MARC records from an alternate MARC server) and metadata.json
 * images/: one image file for each page listed in the IIIF Manifest

## Solr core

In order to support the automatic creation of the solr core, you need to run in "solr cloud" mode. Run solr at startup with:

    ./bin/solr start -c

or use the included docker compose file which will do it for you:

    docker compose up solr

If you use docker compose, ensure that you set `FTEMPO_SOLR_DATA_DIR` in `.env` for the docker volume for the solr core.

Create the solr cores with

    python solr_manage.py create http://localhost:8983 ./solr

where `http://localhost:8983` is the location of the solr instance, and `./solr` is the directory which contains the 
core definition files.

To delete all documents in a core, use

    python solr_manage.py delete_data http://localhost:8983/solr/ftempo

## Metadata loading

We load data into two cores: `ftempo-meta` contains textual metadata from the data download step. `ftempo` contains the data for the music search (MAWs)

    python import_metadata.py ../data-download/GB-Lbl GB-Lbl http://localhost:8983/solr/ftempo-meta

This file will load specific metadata from the library based on what we know we downloaded (currently only implemented
for GB-Lbl and D-MBS)

## Music loading

- **`import_notes.py`** - Processes MEI files generated by OMR to extract musical notes and intervals, then imports this musical data into Solr for search and analysis.

## Image Processing & OMR
- **`omr_pages.py`** - Orchestrates the Optical Music Recognition process using Aruspix. Can process single images or entire directories of book images, generating MEI files containing musical notation data.


## Typical Pipeline Flow

1. **Data Discovery**: Use `download_library.py` to download IIIF manifests and images
2. **Schema Setup**: Use `create_solr_schema.py` to configure Solr cores
3. **OMR Processing**: Use `omr_pages.py` to generate MEI files from images
4. **Metadata Import**: Use `import_metadata.py` to index bibliographic data
5. **Musical Data Import**: Use `import_notes.py` to index musical content
